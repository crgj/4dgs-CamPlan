# 4DGS / 3DGS 拍摄采集领域知识

本技能提供规划拍摄系统所需的领域事实与经验法则，供 `sim/`（覆盖/重叠/曝光/清单）与 `export/`（transforms.json/COLMAP）实现时参考。

## 1. 高斯泼溅重建对采集的要求（经验法则）

- **覆盖度**：主体的可见表面应被多个视角覆盖。静态 3DGS 常见目标：每个表面点被 **≥ 20–50 张**图覆盖；运动/动态区域(4DGS)要求更高且需**时间维度**上的多视角。
- **重叠率**：相邻视角间应有显著共同视野。常见准则：相邻相机**视场重叠 ≥ 60–70%**。重叠不足会导致 SfM 连接断裂、点云稀疏区空洞。
- **baseline（基线）**：相邻相机间距不宜过大（否则特征匹配失败），也不宜过小（否则三角化精度差）。经验：相邻相机间距约为**到主体距离的 1/10 ~ 1/5**。
- **视角多样性**：环绕 + 不同高度，避免纯共面/共线轨迹（否则 SfM 退化，典型退化：纯平移共面 → 退化解）。
- **分辨率**：主体在画面中应占足够像素；经验：主体最小特征 ≥ 数十像素，否则纹理不足。
- **曝光**：避免过曝高光区（高斯在该区域无法重建）与欠曝暗区（噪声主导）。多相机阵列应**曝光/白平衡一致**（v1 至少支持锁定同一组参数并给出一致性警告）。

## 2. v1 仿真输出的关键指标（即 `sim/` 要计算的）

- **逐体素/逐面覆盖计数**：把主体表面离散化（栅格或三角面采样），统计每个采样点被多少相机视锥包含且未被遮挡。→ 输出**覆盖热图**与**最小覆盖数**。
- **盲区**：覆盖计数为 0 的表面区域占比与位置（红色高亮）。
- **欠覆盖区**：覆盖数低于阈值的区域占比。
- **重叠率矩阵 / 平均重叠**：对相邻相机对计算视场内共同可见采样点比例。
- **平均 baseline**：所有相邻相机对的距离统计。
- **曝光一致性**：对每个相机给出的曝光值，统计阵列内方差/极差，超阈值告警。

## 3. 视锥与可见性（`sim/coverage.ts` 实现要点）

- 用纯矩阵构造投影-视矩阵，将世界空间采样点变换到每个相机的裁剪空间：`x,y ∈ [-1,1], z ∈ [near,far]` 且 `z>0` 视为在视锥内。
- **遮挡**：v1 用保守近似——若采样点在主体 AABB 内部则视为被遮挡（不精确，但对“外侧表面”覆盖评估足够）。完整遮挡需光线投射，列为 v2。在代码注释里明确这是近似。
- 栅格化：按主体 AABB 的最长边取 N（默认 256）个采样点/体素，结果分辨率可配；性能与 `N³` 相关，注意缓存。
- 角度用度输入、内部转弧度，统一在 `lib/math` 处理。

## 4. 导出格式（`export/`）

### Nerfstudio `transforms.json`（v1 主格式）
关键字段：
- `camera_model`: `"PERSPECTIVE"`
- `frames[]`: 每帧 `{ file_path, camera_to_world (4x4, row-major), fl_x/fl_y, cx/cy, w/h, ... }`
- 顶层相机参数 `fl_x/fl_y/cx/cy/w/h`（单相机阵列可共享内参）。
- 注意 `camera_to_world` 是**列主序还是行主序**要在导出与校验中固定并注释清楚（Nerfstudio 用 3×4/4x4 行主序存储但 Three.js 矩阵列主序——**这是常见坑**，在 `lib/math` 做一次明确转置）。

### COLMAP (`cameras.txt` / `images.txt` / `points3D.txt`)
- `cameras.txt`: `CAMERA_ID MODEL WIDTH HEIGHT PARAMS...`，MODEL 用 `PINHOLE` 或 `SIMPLE_PINHOLE`。
- `images.txt`: `IMAGE_ID QW QX QY QZ TX TY TZ CAMERA_ID NAME`（四元数 + 平移，世界到相机）。
- 注意 COLMAP 用**世界→相机**变换，而规划器存的是**相机→世界**，导出时取逆；四元数约定为 `w,x,y,z`。在代码注释与单测里固化。

### 自校验（强制）
导出后必须：
1. schema 校验（字段齐全、类型对、值域合理）。
2. round-trip：导出 → 重新解析 → 与原 `SceneDef` 相机集结构相等（允许浮点容差）。

## 5. 4DGS 相对 3DGS 的增量（v1 暂不实现，但接口预留）

- **时间维度**：每个相机姿态可带 `t`（时间戳），阵列 = 多相机 × 多时刻。
- **动态主体**：可在不同时刻有不同 `SubjectDef` 形态（v1 主体静态，但 `CameraDef`/`SceneDef` 已留 `time` 字段位，用可选字段，避免 v2 破坏性改动）。
- **同步要求**：多相机需**硬件/时间同步**，v1 在导出清单里标注每相机的期望触发时刻。

## 6. 当本技能触发时你应该做的事

1. 在写 `sim/` 算法前确认指标定义与本文件一致；有取舍时在代码与 `LOG.md` 记 `DECISION`。
2. 在写 `export/` 时严格按第 4 节格式，并写自校验单测。
3. 注意 4DGS 的时间维度接口预留（第 5 节），不要把 `SceneDef` 写死成纯静态。
4. 数值经验法则是“起点”不是“硬约束”；把它们做成 UI 里**可配置的阈值**，默认值用本文件的推荐值。
